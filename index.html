Project Idea

Overview

This project would consist of the construction of a synthesiser. The synthesiser would have an analogue interface with digital elements. This project would consist of either building a synthesiser from very basic components or more digital and code centric pieces.

Motivation

The motivation for this project comes from an aforementioned interest in music and the desire to build gear. It would certainly be interesting as it encompasses many facets of Information Technology with both connecting hardware through digital and analogue means as well as programming sounds and functionalities into the device and on top of this it can be done many different ways. With cheaper Arduino and raspberry pi powered solutions being an easy way to implement this project being one of many ways of executing it; the concept has the flexibility to be more coding dependant than hardware or to do the opposite.

Description

The synth will need a case fabricated, bare PCB is a bad look as well as dangerous in general for the PCB and the user. The ideal method of input would be for a keyboard style input common among synthesisers with a possibility of drum pads or switches for additional effects, this is what’s known as the controller of the synthesiser. There are additional effects that need to be implemented and modifiable in the final product such as octave, envelopes are what controls the attack decay sustain and release of the synthesizer, attack is the initial part of a note which then lowers down through the decay section, if the note is held it enters into sustain and once it is let go release happens, these components are really important. Another very important component to a synthesiser is a Voltage controlled oscillator which is what produces the wave making it essentially the source of the sound for the synthesiser pair this with the envelope generator and voltage-controlled filters to help shape the sound this is all fed into a voltage-controlled amplifier before reaching the onboard amplifier. In short, the Oscillator makes the noise, the filter shapes it, the envelope changes how it interacts with the piano keys, the voltage-controlled amp which acts as a preamp for the onboard amp. The term voltage control has been mentioned a couple of times and it simply means the sound is adjustable at any point that is voltage controlled which allows for great variability in sound. All the previously listed components allow us to produce the following features:

  1. Different shaped sound waves from an oscillator.
  2. Customisation of Attack, Decay, Sustain and Release.
  3. A set of keys to control the synthesiser.
  4. Ability to move up and down octaves.
  5. A case to house the synth.
  6. Filters to change and shape the oscillators wave.
  7. The amplifiers to make some noise.

All these components are the bare necessities to host said features but there are a few nice to have features such as MIDI support allowing for the use as a MIDI controller with a Digital Audio Workstation. Loop functionality would also be a good feature alongside an arpeggiator function.

Tools and Technologies

Depending on the complexity of the approach there are roughly two different streams. The first option is to use a raspberry pi, Arduino or another board-based technology similar to the two and equip it with some inputs and outputs, this would mean a mostly digital synthesiser that would also be compact. On the other hand, there is the mostly analog option of making a lot of the parts and functions mostly hardware based, this appears to not be a feasible nor as efficient as the previously mentioned board-based method of creation.

Skills Required

For the more likely digital based solution the team working on the project would need to research and select languages to write and develop the code in, this may be limited by the hardware used such as having to only use select languages with Arduino’s due to compatibility with the Arduino board, additionally some fabrication and electronics knowledge will be necessary to manufacture housing and install inputs for the synthesiser.

Outcome

The ideal end product is a functioning and usable synthesiser with standard features built with in expensive resources to a decently functional standard and relatively compact form factor, with a clear and repeatable process as well as a large amount of customisation around codable elements.
